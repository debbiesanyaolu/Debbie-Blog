Unit 2 - Outcome 1 - 1.5 Describing copyright and other constraints affecting websites

A main restriction when making a website is copyright compliance. It protects your work as well as others when you make the following:

  • Originally literary, musical, dramatic and artistic work including illustration and photography.
  • Sound and music recordings.
  • The layout of published editions of written, dramatic and musical works.
  • Broadcasts
  • Original non-literary written work, such as software, web content and databases
  • Film and television recordings

Copyright protects your work from the following:

  • Putting it on the internet
  • Copying your work
  • Distributing copies of it, whether free of charge or for sale
  • Performing, showing or playing your work in public
  • Making an adaptation of your work
  • Renting or lending copies of your work [11]


Unit 2 - Outcome 1 - 1.6 Describing access issues that need taking into account - describe issues that people with disabilities may face when using web pages and how we can overcome them

There are many ways in which you could overcome issues that people with disabilities may face when using the web, these include:

  • Use of alt tags - When you hover your mouse over an image on a website, the little words that pop up are called alt tags. For someone who has a visual impairment and uses a screen reader (a software program that reads text on a website out loud), the alt tags are read aloud, and are the only way a user knows what the image is.
  • Create subtitles and transcripts - If your web content regularly includes videos, try to provide subtitles, especially if you’re producing the bulk of your own video content.
  • Put periods in abbreviations - If you're abbreviating something in HTML, put periods in between each letter.
  • Describe your links - When embedding a link in a post, it's more useful to describe the link, rather than just telling the reader to "click here."
  • Utilize colour control - Practicing smart colour choices is useful for a website with any kind of audience. Avoid pairing garish colors, and be wary of using yellow, blue and green close to one another (this is especially difficult for colorblind users). Dark colour text on a white background is the best general practice, because it's readable for most audiences. [12]


Unit 2 - Outcome 1 - 1.7 Describe which are the appropriate file types for website.

There are many different file types that correspond to making a website with different coding languages, each of which have different functions when making a website. The most frequently used include:

  • JavaScript - It is used primarily for programming a web browser with static HTML and CSS interactive. [13]
  • HTML - HyperText Markup Language is the method by which you move around on the web — by clicking on special text called hyperlinks which bring you to the next page. [14]
  • CSS - Cascading StyleSheet describes how HTML elements are to be displayed.
  • JPEG - Joint Photographic Experts Group file - are image files mainly used for photos
